Aims and Goals

Project Aim

We will demonstrate significant progress towards a set of tangible artefacts that when completed will succeed in creating and developing a device that can independently function as a food scanning device, that is able to scan a product visually through machine learning on Tensorflow, and be able to transmit this information into database over the web.


Goals

Design

he entire goal of this project is to create a device that can independently function as a food scanning device that is able to scan a product visually through machine learning on Tensorflow and be able to transmit this information into database over the web. It will be doing this through a camera on the device placed horizontally, hence items will be scanned while they are lying on a flat surface and from the top looking down. Because of this, it needs to have the capability of attaching itself to a range hood or any type of surface hanging, so making sure that the product is as light as possible are required components of the design. The shells encasing the device must also vary for both visual and functional variation.

Hardware

Research and develop an effective, low-cost hardware solution that is easy to manufacture. Select the right hardware that will be capable of the tasks will be asking of it (image capture, processing and databasing) in a timely matter. Have a list of components needed and drawings of how these components will work together cohesively.

Image Capture

To develop a manual image capture system that will take a comprehensive and high-quality picture every time the end-user interacts with it so that it will have an accurate interaction with TensorFlow. This will be base model of the artefact with plans to integrate further technologies including motion sensor and voice detection technology.

Machine Learning and Visual Recognition

The object recognition component is the intelligence behind the project. Taking the inputs from the camera via the hardware and scripting, the captured image is sent to Tensorflow Lite for processing against a trained model, with the object identification tag returned to the Pi. This is then delivered to the database as the output.

Database

The desired outcome for the databases used in the S.K.I.M (Smart Kitchen Inventory Manager) project is to have multiple databases that can be addressed by other sub systems and to have the databases integrate with each other seamlessly. We need to develop systems that efficiently handle the inputs and outputs of the system, with attention to data security too.

We aim for the databases to:

  • Translate data from the image recognition into labels that the user will understand
  • Store data safely and efficiently, using a method that makes it straightforward to utilise the data in a web application
  • Maintain the feature of scalability so that future development can adapt database processes flexibly

Marketing

The main goal of this product is to help regular households have a much easier time when it comes to inventory management for their produce and food. In turn, this will help them have an easier time keep track of what they need and don’t need whenever they go the grocery store. Because of this, the marketing plan for this product will focus on real life situations and examples that are relatable to the customer, and this would help them see the value they could gain from using the product.

The goal for this artefact is to create a model with strong accuracy to be applied to TensorFlow Lite, and to accomplish this, the following goals must be achieved:

  • Training and test images for food items need to be obtained
  • Training images are imported to the model to “learn” what the item looks like
  • Test images are introduced to the model for identification and confirmation
  • The model will take the identification feedback to enhance its recognition capabilities
  • An accuracy rate as close to 100% as possible
